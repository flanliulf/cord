# v0.2 Performance and Output Governance Plan

Date: 2026-05-21

Purpose: close CR TODOs whose correct resolution is an interface-level or output-contract design, not a small v0.1 patch.

## Scope

- TODO-018: async / batched framework document discovery.
- TODO-023: no-change scan fast-return without full content hashing.
- TODO-027: impact result path explanation for multi-hop hits.

## Decisions

1. `IFrameworkAdapter.discoverDocuments()` remains synchronous for v0.1. v0.2 should design an async discovery API as an explicit interface migration, including batching, directory budgets, cancellation/progress hooks, and compatibility for existing adapters.
2. `ScanService` no-change fast-return remains content-hash based for v0.1. v0.2 should introduce a lazy-hash strategy: compare stored sync state against file stat data first, hash only candidate-changed files, and benchmark large unchanged repositories.
3. `ImpactService` keeps the current compact DTO in v0.1. v0.2 should design a backwards-compatible path explanation extension for CLI JSON and MCP output, including relation chain, intermediate document paths, hop metadata, and candidate tie-break semantics.
4. Timing-ratio assertions do not belong in the main unit suite when they are environment-sensitive. Unit tests should prefer deterministic hot-path evidence such as bounded repository reads and SQLite query-plan/index assertions; wall-clock benchmarks can live in a separate benchmark workflow.

## v0.2 Acceptance Sketch

- Async discovery: a large nested fixture can be scanned without blocking the event loop for the whole traversal, and adapters can opt in without breaking existing sync adapters.
- Lazy hash: unchanged large fixture avoids full file reads/hashes before deciding fast-return; changed-file behavior remains correct.
- Impact explanation: a multi-hop impact result can explain why a document was reached without changing existing required v0.1 fields.
- Performance governance: CI unit tests assert deterministic access patterns; optional benchmarks report timing without becoming flaky release blockers.
